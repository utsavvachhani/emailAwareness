# CyberShield Guard — Technical Workflow & Architecture

This document provides a comprehensive overview of the application's file structure, technology stack, role-based infrastructure, and critical workflows. It is meant to serve as a high-level guide to how the entire system communicates smoothly end-to-end.

---

## 1. Technology Stack

CyberShield Guard is built utilizing a modern, high-performance web architecture:

### Frontend
- **Framework**: Next.js 14+ (App Router)
- **Language**: TypeScript
- **State Management**: Redux Toolkit (Session caching)
- **Styling**: TailwindCSS & Framer Motion (Glassmorphism UI, Dark Mode layouts)
- **Payments**: Stripe Checkout Gateway (`@stripe/stripe-js`)
- **PDF Generation**: `jspdf` & `html2canvas` for professional receipt and certificate rendering

### Backend
- **Environment**: Node.js & Express.js
- **Database**: PostgreSQL (Relational schema via `pg` driver)
- **Authentication**: JWT (JSON Web Tokens)
- **Media Hosting**: Cloudinary (for videos, module attachments)
- **Security**: Helmet, CORS, Express-Rate-Limit

## 3. Platform Portals & Logic

The system strictly utilizes robust Role-Based Access Control (RBAC). 

### 🛡️ Superadmin Portal
**The god-mode command center utilized by platform owners.**
- **Features**: Generates global course curriculums, manages corporate licensing, oversees platform utilization across all registered entities.
- **Workflow**: Has full CRUD (Create, Read, Update, Delete) capability scoped to the database broadly without tenant restrictions.

### 🏢 Admin Portal
**The company management suite.**
- **Features**: Dashboard specifically scaled for HR/IT heads to oversee their internal employees.
- **Billing Modules**: Secure integration mapping subscription tiers (Basic, Standard, Premium) through the Stripe Web UI.
- **Analytics**: Displays live security posture, phishing simulation performance, and awareness risk scores of their teams.

### 🧑‍💻 User Portal (Employees)
**The consumer learning ecosystem.**
- **Features**: Consumes video modules and written documents generated by the superadmin.
- **Assessment Engine**: Users must undergo quizzes to prove feature competence. 
- **Certification Gate**: Certificates are dynamically drafted and unlocked _only_ upon verifying a score threshold (e.g., minimum 3/5 passing grade).

## 4. Key Background Workflows

### The Stripe Payment Workflow (Seamless End-to-End)
This demonstrates how our system flawlessly routes the payment engine smoothly:
1. **Initiation**: The Admin clicks "Authorize & Pay" on the `/admin/.../bills/payment` page.
2. **API Generation**: Front end triggers `POST /api/admin/companies/:id/create-checkout-session` passing their JWT. 
3. **Session Builder**: The backend creates a secure Stripe checkout session calculating real-time pricing tiers and dynamically assigning metadata tracking objects (`company_id`).
4. **Direct Bridge**: The backend returns the `session.url`. The browser natively teleports the user into the encrypted Stripe UI.
5. **Authorization**: User punches in card details (e.g., `4242...`) and Stripe confirms the funds instantly.
6. **Webhook Fire**: Stripe stealthily hits our backend express pathway (`/api/webhook`). This pathway natively bypasses the `express.json()` middlewear to conduct strict SHA-256 signature security checks against the `STRIPE_WEBHOOK_SECRET`.
7. **Fulfillment**: If valid, the SQL Database receives an automatic update (`UPDATE companies SET is_paid = true`).
8. **Professional Rendering**: The UI redirects backwards with `?success=true`. The frontend validates the event, animates a beautiful receipt rendering, auto-triggers a backend email dispatch of the invoice, and unlocks all Premium UI components globally. 

*(Everything flows smoothly and independently synchronously!)*
